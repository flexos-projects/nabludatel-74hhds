# Anomalies Collection

## Purpose

The core output of the system. Stores the structured violations found by the Anthropic API, including the AI's reasoning and the exact trigger quotes.

<flex_block id="blk-001" type="data-schema">
collection: anomalies
fields:
  - name: id
    type: string
    required: true
    description: "Unique UUID for the anomaly"
  - name: section_id
    type: string
    required: true
    description: "Foreign key to sections.section_id"
    references: sections
  - name: severity
    type: enum
    required: true
    description: "Classification of the violation"
    enum_values: [warning, critical]
  - name: category
    type: string
    required: true
    description: "Typology (e.g., 'Intimidation', 'Protocol Falsification')"
  - name: timestamp
    type: string
    required: true
    description: "The exact time in the video (e.g., '01:45:22')"
  - name: trigger_quote
    type: string
    required: true
    description: "The exact spoken words from the transcript that triggered the alert"
  - name: ai_reasoning
    type: string
    required: true
    description: "Claude's explanation for why this is a violation"
  - name: review_status
    type: enum
    required: true
    description: "Human verification state"
    enum_values: [unreviewed, verified, false_positive]
    default: unreviewed
  - name: created_at
    type: date
    required: true
    description: "When the anomaly was found"
indexes:
  - name: idx_anomalies_severity
    fields: [severity]
    description: "Sorting the feed"
  - name: idx_anomalies_review_status
    fields: [review_status]
    description: "Filtering out false positives from the feed"
  - name: idx_anomalies_created_at
    fields: [created_at]
    description: "Reverse-chronological sorting for the feed"
</flex_block>

## Access Patterns

| Operation | Who | When | Frequency |
|---|---|---|---|
| Create multiple | Orchestrator | After Claude analysis | Medium |
| Read latest | Dashboard API | Polling the Anomaly Feed | High (every 5s) |
| Update `review_status`| Dashboard API | User clicks Mark Verified / False Positive | Low |

## Relationships

- `section_id` → `sections.section_id` (N:1 relationship)

## Constraints

- `id` must be unique.
