Route: /section/:id
Prototype: section-detail-v1.html (pending generation)

Key decisions:
- Layout: 600px wide right-anchored slide-over panel.
- Primary data source: MOCK_DB.sections, MOCK_DB.anomalies, MOCK_DB.transcripts
- Main interaction: Clicking "Watch Video" opens evideo.bg in a new tab.
- Visuals: Feels like a physical evidence folder. Dense text, clear highlights on trigger quotes.
</flex_block>

## Page Planning
1. **Who arrives here?** A user who just clicked an anomaly card in the feed. They are skeptical and need evidence to verify the AI's claim.
2. **What story does this page tell?** "Here is exactly where and when it happened → Here is why the AI flagged it → Here is the surrounding context → Go watch it yourself."
3. **What's the conversion/action goal?** User clicks "Watch Video @ [Timestamp]" to verify the footage.
4. **What's the UX vision?** A forensic dossier. Heavy, opaque panel. Clear, unambiguous action buttons.
5. **What sections create that story?** Header (metadata), AI Reasoning Block, Contextual Transcript, Action Bar.
6. **What would make someone call someone over to look at this?** The direct connection between the AI's reasoning and the highlighted raw transcript, proving the system's exactness.

## Purpose
Provide the full context needed to verify a single anomaly without losing sight of the main feed.

## Layout Shell
- Shell: `overlay` (Slide-over panel)
- Sidebar: `no`
- Topbar: `no`
- Wrapper classes: `.slide-over`, `.dossier-panel`

## Sections (top to bottom)

### Header
- **Wrapper:** `.dossier-header`
- **Data:** `MOCK_DB.sections` → [Section ID, Municipality, Village name], `MOCK_DB.anomalies` → [Severity tag]
- **Copy:** `dynamic: section_id`, `dynamic: settlement`, `dynamic: severity`
- **Components used:** `.tag-severity`, `.mono-heading`
- **Notes:** High contrast. Shows exactly where this is happening.

### AI Reasoning Block
- **Wrapper:** `.reasoning-block`
- **Data:** `MOCK_DB.anomalies` → [ai_reasoning]
- **Copy:** `dynamic: ai_reasoning`
- **Components used:** `.json-render`, `.text-sans`
- **Notes:** Rendered readably but clearly generated by AI.

### Contextual Transcript
- **Wrapper:** `.transcript-context`
- **Data:** `MOCK_DB.transcripts` → [content_json]
- **Copy:** `dynamic: transcript_snippet`
- **Components used:** `.mono-block`, `.highlight-trigger`
- **Notes:** Shows 5 minutes before and after. The exact `trigger_quote` is highlighted with a stark background color.

### Action Bar (Sticky bottom)
- **Wrapper:** `.action-bar-sticky`
- **Data:** `MOCK_DB.anomalies` → [timestamp]
- **Copy:** "Watch Video @ [Timestamp]", "Mark as False Positive", "Mark Verified"
- **Components used:** `.btn-primary`, `.btn-massive`, `.btn-secondary`
- **Notes:** Massive primary button for watching the video. Secondary buttons for triage status.

## Interactive States

### Triage Action (Verify / False Positive)
- **Trigger:** User clicks "Mark Verified" or "Mark as False Positive"
- **Vue ref:** `const reviewStatus = ref('unreviewed')`
- **Implementation:**
  ```js
  const markStatus = (status) => {
    reviewStatus.value = status;
    // Show toast
    showToast(status === 'verified' ? 'Anomaly verified. Logged for export.' : 'Marked as False Positive. Removed from active triage.');
    // Close overlay
    closeOverlay();
  };
  ```
- **Before:** Buttons active
- **After:** Toast appears, overlay slides out, feed updates (card removed or marked).

## Responsive
**Mobile (320-639px):**
- Overlay becomes a full-screen modal. Action bar remains sticky at bottom.

- Overlay takes up 80% of screen width.

- 600px fixed width, anchored to the right edge. Left side of screen (dashboard feed) remains visible and dimmed.
