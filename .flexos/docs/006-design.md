# Design

## Design Direction
Nabludatel is a forensic tool built for the night shift. It borrows from the visual language of terminals, air traffic control, and financial trading platforms (think "Bloomberg Terminal meets Palantir"). Information density is extremely high. Contrast is stark. It explicitly avoids friendly consumer UI patterns—no soft shadows, no playful illustrations, no rounded, bubbly buttons. The aesthetic is clinical, objective, and unbribable. Data is the only decoration.

## UX Patterns
**The Live Telemetry Pulse**
The sidebar isn't just static numbers. It features a small, pulsing indicator (a pure red dot) that signals the pipeline is actively downloading and processing. When numbers increment (e.g., Sections Processed), they don't just snap to the new value; they tick over like a physical counter. This reassures the tired user that the machine is working.

**The Urgent Arrival (Anomaly Feed)**
When a new anomaly is found, it does not fade in softly. It snaps to the top of the feed with a hard, high-contrast flash (a 1-second border highlight) before settling into its card state. It demands immediate attention.

**Context Preservation (The Dossier Slide-Over)**
Clicking an anomaly never navigates away from the dashboard. It slides a heavy, opaque panel over the right side of the screen. The user can still see the live feed ticking on the left. The slide-over feels like opening a physical evidence folder—dense text, clear highlights on the trigger quotes, and massive, unambiguous action buttons.

## Component Patterns
**Anomaly Cards**
Strictly rectangular. Left-bordered with a thick, solid color indicating severity (Amber for warnings like delays; Crimson for criticals like intimidation). The header contains monospace metadata (Section ID, Time). The body is sans-serif for readability, but the transcript snippet is always rendered in a dark, inset monospace block to look like raw code/data.

**Coverage Matrix Rows**
Hyper-compressed table rows. No padding. Statuses are indicated by small, solid geometric shapes (square for complete, triangle for flagged, circle for pending) rather than text badges, allowing the user to scan thousands of rows visually.

## Layout Strategy
**Desktop-Only App Shell**
This is not a mobile app. It is designed for large external monitors in newsrooms or observer command centers. 
- **Left:** Fixed 280px sidebar for telemetry and system status.
- **Center/Right:** Fluid width feed.
- **Overlay:** 600px wide right-anchored slide-over for section details.

## Motion & Animation Language
Motion is restricted entirely to communicating state changes. 
- **The Pulse:** A sharp, 2-second heartbeat animation on the live status dot.
- **The Arrival:** 0ms ease-in, 500ms ease-out flash when a new alert drops in.
- **The Slide-Over:** Fast, linear translation (150ms). No bouncy spring physics. It is a machine delivering a report.

## Microcopy Tone
The product speaks in facts, numbers, and timestamps. It has zero personality. It is objective and clinical.
- *Error:* "DATABASE CONNECTION LOST. Retrying in 5s." (Not: "Oops! We can't reach the database.")
- *Empty State:* "Pipeline active. 1,204 sections queued. 0 anomalies detected." (Not: "You're all caught up! Go grab a coffee.")
- *Confirmation:* "Marked as False Positive. Removed from active queue."

## Empty States
**The Initial Dashboard**
- Headline: `SYSTEM INITIALIZED`
- Description: `Waiting for evideo.bg archive to populate. Polling every 60 seconds.`
- Visual: A blinking monospace cursor `_`

## Colour Direction
Anchored in absolute dark mode. The background is a dense charcoal, allowing the stark white text and aggressive accent colors to pop. Primary text is crisp. Accents are purely functional: Amber for warnings, Crimson for critical alerts. There is a light mode for daytime review, but it feels like a printed forensic report (white paper, black ink).

## Typography Direction
Typography is the primary UI element. Monospace is used for all metadata, timestamps, and section IDs to ensure perfect vertical alignment in tables and feeds. A highly legible sans-serif is used for the actual AI reasoning and transcript body text to prevent reading fatigue.

## Iconography
Lucide icons. 1.5px stroke weight. Sharp, unrounded edges where possible. Used sparingly—only when they convey status faster than text.
