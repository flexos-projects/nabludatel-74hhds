# Database

## Stack Note
**Database:** SQLite.
**Why:** Zero-configuration, local, bulletproof state management. It acts as an unbreakable ledger for the 12,000 records. If the power goes out, SQLite ensures the system boots back up exactly where it left off. No cloud sync, no latency.

## Position/Computed Fields
- `Dashboard Stats`: The telemetry sidebar (Sections Queued, Processed, Critical Alerts) is computed by counting rows in `processing_queue` by `status`, and `anomalies` by `severity`. These should be queried efficiently or materialized via a view.

## File Storage
- **Ephemeral Video:** `.mp4` files are downloaded to a local `/tmp/videos` directory. Maximum size ~2GB per file. **Must be deleted immediately** after audio extraction.
- **Ephemeral Audio:** `.wav` files are stored in `/tmp/audio`. **Must be deleted immediately** after Whisper transcription completes.
- **Persistent Storage:** Only the SQLite database file (`nabludatel.db`) is kept permanently.
