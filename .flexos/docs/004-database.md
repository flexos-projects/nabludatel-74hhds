# Database

## Stack Note
**Database:** SQLite.
**Why:** Zero-configuration, local, bulletproof state management. It acts as an unbreakable ledger for the 12,000 records. If the power goes out, SQLite ensures the system boots back up exactly where it left off. No cloud sync, no latency.

**Concurrency Configuration:** To support simultaneous reads from the Express dashboard and heavy writes from the Orchestrator, `better-sqlite3` MUST be configured with Write-Ahead Logging (`PRAGMA journal_mode = WAL;`) and a busy timeout. See `specs/009-feature-primary_orchestrator.md`.

## Collections / Tables

The detailed data schemas for each collection are defined in their respective spec files:

- **Sections** (`specs/004-database-collection_sections.md`): The master list of all 12,000+ polling stations in Bulgaria. Pre-seeded before the pipeline starts.
- **Processing Queue** (`specs/005-database-collection_processing-queue.md`): The state machine tracking each section's journey through the pipeline.
- **Transcripts** (`specs/006-database-collection_transcripts.md`): The raw output from faster-whisper. Stored as JSON arrays of timestamped text blocks.
- **Anomalies** (`specs/007-database-collection_anomalies.md`): Alerts generated by Claude. The core output of the system.

## Position/Computed Fields
- `Dashboard Stats`: The telemetry sidebar (Sections Queued, Processed, Critical Alerts) is computed by counting rows in `processing_queue` by `status`, and `anomalies` by `severity`. These should be queried efficiently or materialized via a view.

## File Storage
- **Ephemeral Video:** `.mp4` files are downloaded to a local `/tmp/videos` directory. Maximum size ~2GB per file. **Must be deleted immediately** after audio extraction.
- **Ephemeral Audio:** `.wav` files are stored in `/tmp/audio`. **Must be deleted immediately** after Whisper transcription completes.
- **Persistent Storage:** Only the SQLite database file (`nabludatel.db`) is kept permanently.
