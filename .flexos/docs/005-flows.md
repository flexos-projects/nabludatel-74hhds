# Flows

## Core Flows

### 1. The Night-Shift Pipeline (System Flow)
- **Flow name:** Autonomous Pipeline Execution
- **Actor:** Python/Node Orchestrator Script
- **Sequence:**
  1. → Query `processing_queue` for next section where `status = pending`, ordered by `Sections.is_rural DESC`.
  2. → Update `status = downloading`.
  3. → Download `.mp4` from `evideo.bg`.
  4. → Extract audio to `.wav` and **DELETE** `.mp4`.
  5. → Update `status = transcribing`.
  6. → Run `faster-whisper` locally on `.wav` to generate JSON transcript.
  7. → **DELETE** `.wav`.
  8. → Update `status = analyzing`.
  9. → Send transcript to Claude CLI (`claude -p`) with forensic prompt.
  10. → Parse Claude's JSON response.
  11. → If anomalies found, write to `anomalies` table.
  12. → Write transcript to `transcripts` table.
  13. → Update `status = complete`.
- **Edge cases:** If download fails (network drop), catch exception, increment `retry_count`, set `status = pending`. If Claude API times out, set `status = transcribing` (so it retries analysis without re-transcribing).

### 2. Triage and Verification (User Flow)
- **Flow name:** Anomaly Triage
- **Actor:** Human Observer / Journalist
- **Sequence:**
  1. → User stares at Command Dashboard (`/`).
  2. → New alert card flashes into the Anomaly Feed.
  3. → User clicks the card.
  4. → Section Detail overlay slides in.
  5. → User reads Claude's reasoning and the surrounding transcript snippet.
  6. → User decides if it warrants investigation.
  7. → User clicks "Watch Video @ [Time]".
  8. → New browser tab opens to `evideo.bg` at the exact timestamp.
  9. → User watches the footage.
  10. → User returns to dashboard and clicks either "Mark Verified" or "Mark False Positive".
- **Decision points:** 
  - Is it a real violation? → Mark Verified.
  - Is it AI hallucination/misunderstanding? → Mark False Positive (updates `anomalies.review_status`).

## State Machines

### Processing Queue State Machine
Entity: `processing_queue` record.
- `pending` → (Script picks up) → `downloading`
- `downloading` → (Success) → `transcribing`
- `downloading` → (Network Error) → `pending` (increments retry)
- `transcribing` → (Success) → `analyzing`
- `transcribing` → (Hardware Error) → `failed`
- `analyzing` → (Success) → `complete`
- `analyzing` → (API Timeout) → `transcribing` (to retry analysis)

## Error States
- **Disk Full Error:** If the script detects < 5GB free space, it halts the pipeline gracefully and flashes a critical system alert on the dashboard.
- **Claude Rate Limit:** If the Anthropic API returns a 429, the script sleeps for 60 seconds before retrying the `analyzing` state.
- **Missing Video:** If `evideo.bg` returns a 404 for a section, the script marks it `failed` with error log "404 Not Found" and moves to the next section immediately.
